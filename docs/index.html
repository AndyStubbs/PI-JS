Graphics, Sound, and Input Library

Introduction

QBS is a javascript library that is inspired by QBasic and is intended to help make pixel-perfect retro games and demos in the web browser. QBS API is modelled after QBasic and is easy to use for beginners.

Usage

QBS is a collection of graphics, input, and sound commands that are all stored in a global variable container. QBS commands can be accessed by using the global variable "qbs" or "$". If there is already a global variable set to "$" then the QBS library will not override the "$". This is in case you want to use other libraries such as jQuery that use the "$" global.

Graphics commands require a screen to be created before being called. The screen command creates a canvas object and adds it to the page by default. When the screen command is called a screen object is returned. Graphics commands can be called from either the global "$", "qbs", or the screen object. 

$.screen("300x200");
$.line(10, 10, 290, 190);
qbs.screen("300x200");
qbs.line(10, 10, 290, 190);
var screen = $.screen("300x200");
screen.line(10, 10, 290, 190);

The screen object comes in handy when working with multiple screens. If you are only working with one screen the simpliest way to run the graphics commands is to use the "$" variable. If you are working with multiple screens you can still use "$" variable but you will have to call the setScreen command if you want to change drawing screens.

There are two ways to call commands, the first is to use a comma seperated list of parameters and the second is to pass in an object that contains names of the parameters as object properties. 

$.screen("300x200");

// Comma seperated list call
$.line(10, 10, 290, 190);

// Object properties call
$.line({
	"x1": 10, 
	"y1": 10, 
	"x2": 290, 
	"y2": 190
});
